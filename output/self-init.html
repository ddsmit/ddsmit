Welcome to ddsmit! I plan for this website to serve a couple different purposes:

  1. Act as a Notepad/Reference for any snags I run into (persist)
  2. Document my learning path (self)
    • What I'm learning
    • How I'm learning
    • What I plan to learn
  3. Act as a profile for potential employers (__repr__)
    • Allows deeper insite than a static resume can provide
    • Can tailor view for information an employer values most
    • Allows linking to github and other resources I work on

The "this" series will be a series of blog posts that document building and improving the site. My target is to make a step forward every day, regardless of how small (and how bad it is to start).

__init__(self, day=1, generator=pelican)

I am very new to web development. I built my wifes website Courageous Engineer using Flask (with several extensions) and some Boostrap on the frontend. I roughly followed a combination of Corey Schafer's and Talk Python's tutorials (both are highly recomended).

Using Flask and creating the backend was a fun learning experience, and it definately fit in better with my experience of using Python for data analysis at work. That being said, I struggled A LOT with the HTML and CSS. I was able to get a product I was happy with, but there were some challenges along the way.

For my personal page, I wanted to try to build a static webpage with a static page generator. I really enjoyed using Flask, but I wanted to minimize hosting costs (especially since I don't have any needs for a server right now). Intially, I looked at GatsbyJS (also a reason to force myself to do more JavaScript), but I got a recomendation from Twitter to try Pelican.

Pelican has been interesting so far. I was able to get an OK site up very quickly using the Quickstart Guide (my old computer was by far the bottleneck), but it has definately been a challenge trying to an end product closer to what I want. Often times, I feel like the more abstracted a task becomes, the quicker the execution can be, but the more difficult it is to tweak.

Every tool comes with its own set of challenges, and I'm looking forward to learning more about static site generation and becoming better at HTML, CSS, and JavaScript.
