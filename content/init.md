## \__init__(self, day=1, generator=pelican)

I am very new to web development. I built my wifes website [Courageous Engineer](https://courageousengineer.com) using [Flask](https://flask.palletsprojects.com/en/1.1.x/) (with several extensions) and some Boostrap on the frontend. I roughly followed a combination of [Corey Schafer's](https://www.youtube.com/watch?v=cYWiDiIUxQc&list=PL-osiE80TeTs4UjLw5MM6OjgkjFeUxCYH&index=4) and [Talk Python's](https://training.talkpython.fm/courses/explore_flask/building-data-driven-web-applications-in-python-with-flask-sqlalchemy-and-bootstrap) tutorials (both are highly recomended). 

Using Flask and creating the backend was a fun learning experience, and it definately fit in better with my experience of using Python for data analysis at work. That being said, I struggled __A LOT__ with the HTML and CSS. I was able to get a product I was happy with, but there were some challenges along the way.

For my personal page, I wanted to try to build a static webpage with a static page generator. I really enjoyed using [Flask](https://flask.palletsprojects.com/en/1.1.x/), but I wanted to minimize hosting costs (especially since I don't have any needs for a server right now). Intially, I looked at [GatsbyJS](https://www.gatsbyjs.org/) (also a reason to force myself to do more JavaScript), but I got a recomendation from Twitter to try [Pelican](https://docs.getpelican.com/en/stable/index.html).

[Pelican](https://docs.getpelican.com/en/stable/index.html) has been interesting so far. I was able to get an OK site up very quickly using the [Quickstart Guide](https://docs.getpelican.com/en/stable/quickstart.html) (my old computer was by far the bottleneck), but it has definately been a challenge trying to an end product closer to what I want. Often times, I feel like the more abstracted a task becomes, the quicker the execution can be, but the more difficult it is to tweak.

Every tool comes with its own set of challenges, and I'm looking forward to learning more about static site generation and becoming better at HTML, CSS, and JavaScript.
